### SHA-1 해시로 변경 감지

```swift
# 파일 내용이 같으면 항상 같은 해시
echo "Hello World" | git hash-object --stdin
# 출력: 557db03de997c86a4a028e1ebd3a1ceb225be238

# 내용이 바뀌면 다른 해시
echo "Hello World!" | git hash-object --stdin
# 출력: 980a0d5f19a64b4b30a87d4206aade58726b60e3
```

### Tree 객체 비교

```swift
이전 커밋의 tree:
├── README.md (blob: abc123...)
├── src/
│   └── main.py (blob: def456...)

현재 커밋의 tree:
├── README.md (blob: abc123...)  ← 변경 없음
├── src/
│   └── main.py (blob: xyz789...)  ← 변경됨!
```

## Git

Git은 데이터를 객체(Object) 데이터베이스에 저장하여 버전 관리를 수행합니다.

## Git 객체란?

Git은 파일의 변경 사항이 아닌 콘텐츠 자체를 저장합니다. 모든 객체는 콘텐츠를 기반으로 생성된 40자리의 고유한 SHA-1 해시 값으로 식별됩니다. 이 해시 값은 객체의 이름이자, 객체를 찾기 위한 키(key)로 사용됩니다.

### 블롭 객체(Blob Object)

블롭 객체는 파일 하나의 내용을 저장하는 가장 기본적인 Git 객체입니다.

- 콘텐츠 저장: 블롭은 파일의 실제 데이터를 압축하여 저장합니다. 파일의 이름, 경로, 생성일 등 메타데이터는 포함하지 않고 오직 파일의 내용만을 담습니다.
- 효율적인 저장: 만약 서로 다른 두 파일의 내용이 완전히 같다면, Git은 새로운 블롭 객체를 만들지 않고 기존의 블롭 객체를 공유합니다. 이로 인해 저장소 용량을 효율적으로 관리할 수 있습니다. 예를 들어, `file1.txt`와 `file2.txt`의 내용이 동일하다면 두 파일은 같은 블롭 객체를 가리키게 됩니다.
- 불변성: 블롭 객체는 한 번 생성되면 그 내용이 절대 변경되지 않습니다. 파일 내용이 1바이트라도 수정되면, Git은 완전히 새로운 내용으로 새로운 블롭 객체를 생성합니다.

## 커밋 객체(Commit Object)

커밋 객체는 프로젝트의 특정 시점, 즉 **하나의 스냅샷**을 기록하는 객체입니다. 사용자가 `git commit` 명령을 실행할 때마다 새로운 커밋 객체가 생성됩니다.

- 트리 객체 포인터: 해당 커밋 시점의 프로젝트 전체 디렉터리 구조를 나타내는 **최상위 트리(Tree) 객체**의 SHA-1 해시 값을 가리킵니다. 트리는 특정 시점의 파일과 하위 디렉터리들의 스냅샷 정보를 담고 있으며, 각 파일은 블롭 객체와 연결됩니다.
- 부모 커밋 포인터: 바로 이전 커밋(부모 커밋)의 SHA-1 해시 값을 가리킵니다. 이 연결 고리를 통해 Git은 프로젝트의 전체 히스토리를 추적할 수 있습니다.
  - 첫 번째 커밋(루트 커밋)은 부모가 없습니다.
  - 두 개 이상의 브랜치를 병합(merge)하는 경우, 해당 커밋은 여러 개의 부모 커밋을 가질 수 있습니다.
- 작성자(Author): 코드를 최초로 작성한 사람의 정보(이름, 이메일, 작성 시각)입니다.
- 커미터(Committer): 해당 커밋을 저장소에 기록한 사람의 정보(이름, 이메일, 커밋 시각)입니다. 대부분의 경우 작성자와 커미터는 동일하지만, 다른 사람의 패치를 적용하는 경우 달라질 수 있습니다.
- 커밋 메시지: 해당 커밋의 변경 사항을 설명하는 메시지입니다.
